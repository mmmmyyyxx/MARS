**Enhanced Formal Fallacy Detection with Guided Reasoning and Exemplars:**

"Your task is to determine whether the conclusion **must** be true given the premises, using **only** the explicitly stated information. Follow this structured reasoning process:

**Step 1: Logical Translation Template**
- Translate each premise and conclusion into symbolic form using variables (A, B, C) and quantifiers (all/some/none)
- **Critical for negations:** Carefully distinguish between "not all" (which means "some are not") and "none" (which means "all are not")

**Step 2: Counterexample Construction Template**
- **Systematically test:** Create scenarios where all premises are true but conclusion is false
- **For negation cases:** Specifically test if "not all X are Y" allows for "some X are Y" while conclusion fails
- **Verification question:** "Can I construct at least one logically consistent scenario where premises hold but conclusion fails?"

**Illustrative Examples:**
- **Valid case:** "All A are B. X is A. Therefore X is B." → No counterexample possible
- **Invalid case:** "Not all A are B. X is A. Therefore X is not B." → Counterexample: Some A are B, and X could be one of them

**Step 3: Final Determination**
- If **any possible counterexample** exists → **invalid**
- If **no counterexample** can be constructed → **valid**

**Key Constraints:**
- Use ONLY explicitly stated information
- No real-world knowledge or assumptions
- Focus particularly on proper interpretation of negations

Output only 'valid' or 'invalid' after completing this verification."
